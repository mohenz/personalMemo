# PersonalMemo Mobile Minimum Feature Plan

## 목적

PersonalMemo는 현재 PC 사용자와 태블릿 사용자 환경에 적합한 넓은 화면 중심 UI로 구성되어 있다.
모바일 환경은 전체 기능을 축소해 제공하기보다, 이동 중에도 최소 기능을 빠르게 사용할 수 있는 보조 모드로 설계한다.

모바일 사용자의 핵심 목표는 다음과 같다.

- 오늘 일정 확인
- 새 메모 또는 일정 빠른 등록
- 기존 메모 검색
- 중요한 메모와 일정 열람 및 간단 수정

따라서 모바일 UI는 고급 작성 도구가 아니라 "포켓 대시보드"에 가깝게 기획한다.

## 기본 화면 구조

모바일 첫 화면은 단일 대시보드로 단순화한다.

- 상단: 오늘 날짜, 검색 버튼, 추가 버튼
- 본문 1: 오늘 일정
- 본문 2: 최근 메모
- 하단 탭: 오늘, 메모, 캘린더, 설정

PC/태블릿 UI의 사이드바, 2-pane 메모 목록/상세 분할, 복잡한 자료실 UI는 모바일 첫 화면에서 제외한다.

## 필수 기능 범위

| 영역 | 모바일 기능 | 설명 |
| --- | --- | --- |
| 오늘 | 오늘 일정 보기 | 체크리스트 또는 일정이 있는 메모를 날짜 기준으로 노출한다. |
| 빠른 추가 | 새 일정/메모 등록 | 제목, 날짜, 내용 중심의 간소화 폼을 제공한다. |
| 메모 | 최근 메모 목록 | 제목, 일부 내용, 작성 시간을 카드형 리스트로 표시한다. |
| 검색 | 전체 검색 | 제목, 본문, 일정 텍스트를 검색한다. |
| 상세 보기 | 메모 읽기 | 읽기 중심 화면으로 이미지와 체크리스트를 표시한다. |
| 간단 수정 | 제목/내용/일정 수정 | 전체 편집기보다 단순한 모바일 편집 화면을 제공한다. |
| 캘린더 | 주간 또는 리스트형 캘린더 | 작은 화면에서는 월간 그리드보다 주간/리스트형을 우선한다. |
| 계정 | 로그인/동기화 상태 | Firebase 계정 상태와 저장 상태를 표시한다. |

## 초기 모바일 버전에서 제외할 기능

초기 버전에서는 아래 기능을 숨기거나 읽기 전용으로 둔다.

- 자료실 파일 관리 전체 기능
- 폴더/그룹 고급 관리
- 이미지 다중 첨부와 라이브러리 선택
- 복잡한 편집 도구
- PC형 월간 캘린더 그리드 전체 조작
- 테마와 프로필 상세 설정
- 휴지통/아카이브 고급 관리

필요 시 이후 단계에서 더보기 메뉴나 설정 화면으로 확장한다.

## 권장 모바일 UX

모바일에서는 하단 고정 액션을 중심으로 조작한다.

- 우측 하단 `+` 버튼을 기본 추가 진입점으로 사용한다.
- `+` 버튼 클릭 시 `일정 추가`, `메모 추가`를 선택한다.
- 일정 추가 기본값은 날짜를 오늘로 설정한다.
- 일정 추가 필드는 제목을 필수로 하고, 시간과 내용은 선택으로 둔다.

메모 상세 화면은 다음 조작만 전면에 둔다.

- 뒤로가기
- 수정
- 더보기 메뉴

삭제, 즐겨찾기, 공유 같은 부가 기능은 더보기 메뉴 안으로 이동한다.

## 구현 단계 제안

1. 모바일 감지 레이아웃 추가
   - `sm` 이하 화면에서 모바일 전용 대시보드를 렌더링한다.

2. 모바일 전용 컴포넌트 분리
   - 예: `MobileDashboard`, `MobileNoteList`, `MobileNoteDetail`, `MobileQuickAdd`

3. 기존 데이터 구조 재사용
   - Firebase 저장 구조는 변경하지 않는다.
   - 기존 `notes`, `groups`, `dateString`, `checklist` 구조를 그대로 사용한다.

4. 1차 구현 범위
   - 오늘 일정
   - 최근 메모
   - 검색
   - 상세 보기
   - 빠른 추가

5. 2차 확장 범위
   - 주간 캘린더
   - 간단 수정
   - 이미지 첨부
   - 모바일 설정

## 설계 원칙

모바일 UI는 PC 앱의 축소판으로 만들지 않는다.
사용자가 이동 중에 다음 질문을 빠르게 해결하는 companion UI로 설계한다.

- 오늘 뭐가 있지?
- 지금 떠오른 내용을 어디에 저장하지?
- 예전에 적어둔 메모가 어디 있지?

이 방향이 작은 화면에서 가장 실용적이며, 기존 PC/태블릿 중심 UI와도 충돌하지 않는다.
